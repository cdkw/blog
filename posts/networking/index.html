• IP Addresses - LAYER 3 (in the OSI model)




=——– IPv4 ——–=

*inet => IPv4 (decimal notation x.x.x.x from 1<=x<=255)
where x has 8 bits, so an IP address in binary looks like 00100100.10100011.00100100.10100011 (8bits = 1byte, so an IPv4 address is 4bytes)

How binary works:
128 64 32 16 8 4 2 1
1 1 1 1 1 1 1 1 = 255

NAT (Network address translation):

  • It assigns Private IP Addreses, and only the router has that Public IP Address, got from the ISP. Private IP Addresses begin usually look something like 192.168.0.x, 10.0.0.x, 127.16.x.x-127.31.x.x.




=——– IPv6 ——–=

*inet6 => IPv6 (hexadecimal notation fe80::x:x:x:x from 0000 <= x <= ffff in hexadecimal notation)

IPv6 was invented because there are too many IPv4 addresses, 2^32 ~= 4.3 billion, where there are ~8 billion people on the Earth.
So it was invented to make the more space, because 2^128 is a big number. But it ended up not being used because of NAT (Network address translation)


• MAC Addresses - LAYER 2 (in the OSI model)

MAC (ether) stands for Media Access Control - a “physical address”, a way to communicate if using switches. Every NIC (Network Interface Card) has a MAC address (EX: your phone).

FORMAT: 00:00:00:00:00:00 - ff:ff:ff:ff:ff:ff

They have identifiers, you can use the first 3 pairs (EX: 00:0c:29... ) to identify a device:
https://dnschecker.org/mac-lookup.php and it will end up showing VMWare, Inc..

A modern router is a L2 and L3 device, meaning it’s doing switching and routing for you. So that means it might know the MAC Address


• TCP/UDP - LAYER 4 (transport layer in the OSI model)

UDP - User Datagram Protocol (Streaming service, VoIP, DNS, …)

TCP - Transition Control Protocol (connection oriented):

  • high reliability
  • three way handshake (EX: How you connect to a port on a machine)

SYN > (Hello!) SYN ACK < (Hello back!) ACK < (acknowledgement)


• Three-way handshake

You can scan in nmap with a three-way handshake using the -sT parameter.

TCP

• Telnet (23) - Log into a machine remotely
• SSH (22) - Telnet but encrypted
• SMTP (25) - Mail
• IMAP (143) - Mail
• POP3 (110) - Mail
• DNS (53) - Resolve IP Addresses to names (Domain Name System)
• HTTP (80) / HTTPS (443 - encrypted) - Website

• MySQL (3306) - Database

UDP

• DNS (53) - Resolve IP Addresses to names
• DHCP (67, 68) - Associates you an IP Address (opposite of static IP)
• TFTP (69) - Trivial FTP
• SNMP (161) - Simple Network Management Protocol

• The OSI Model

(Please Do Not Throw Sausage Pizza Away! 😆) LAYER 1. Physical layer - data cables, cat6 LAYER 2. Data layer - switching, MAC addresses LAYER 3. Network - IP Addresses, routing LAYER 4. Transport layer - TCP/UDP LAYER 5. Session - session management LAYER 6. Presentation layer - WMV, JPEG, MOV LAYER 7. Application layer - HTTP, SMTP

• Subnetting (important!)

netmask = subnet = subnet mask netmask 255.255.255.0 (Just like IPv4, x.x.x.x) How 255.255.255.0 looks in binary 11111111.11111111.11111111.00000000

255.255.255.0 is a WACK /24 network, commonly used in houses and small businesses (256 hosts and 254 usable hosts)
Bits get turned on from the left to the right —>

The /24 comes from 11111111.11111111.11111111.00000000 because there are 24 bits turned on. The bigger the company, the bigger the host number might be, so for example instead of /24 a company might have a WACK /16, subnet which looks something like 255.255.0.0 and an IPv4 example being 10.0.1.5

One way to think about it is that the 255 locks a number in place, and a 0 lets it be any number from 0-255:

192.168.110.4
255.255.255.0

That means there are 256-2=254 hosts possible, and they can have IP Addresses in range 192.168.110.1-192.168.110.255.

We subtract 2 from the final hosts because the network has a:

  • Broadcast (which is usually the last address. EX: for 192.168.0.1/24 the broadcast is 192.168.0.255)
  • Network ID (which is usually the first address. EX: for 192.168.0.1/24 the network id is 192.168.0.0)



If you do not have the Cyber Mentor’s Subnetting Sheet with you, an easy way to know the number of hosts if a CIDR number is given is to:

  1. Subtract CIDR value from 32 (Example: /24. 32 - 24 = 8)
  2. Take the difference and use it as power to base 2 (Example: 2^8 = 256)
  3. Subtract 2 from the result to get your number of hosts (Example: 256 - 2 = 254 number of hosts)

192.168.0.0/22 255.255.252.0 1022 192.168.0.0 192.168.3.255
192.168.1.0/26 255.255.255.192 62 192.168.1.0 192.168.1.63
192.168.1.0/27 255.255.255.224 30 192.168.1.0 192.168.1.31
